# Contributing
1. Fork repo.
2. Create feature branch.
3. Add changes + tests.
4. Submit PR.

## Guidelines
- Follow PEP 8.
- Comprehensive tests for new features.
- Update docs for API changes.
- Backward compatibility.

## Git & GitHub
GitHub sole provider. Git sole VCS. 2FA required for accounts. GPG signed PRs preferred.

## Coding Standards
Follow [GNU Coding Standards](https://www.gnu.org/prep/standards/).
